# Shared Jest pool

- Unexpected hurdle: the first launcher integration used a `URL` object where Node expected a filesystem path.
- Diagnosis: the focused wrapper command failed before Jest started; converting the module URL with `fileURLToPath` fixed it.
- Chosen fix: coordinate normal Jest processes with an atomic shared lock and PID-based stale-owner recovery; retain Jest's existing per-process worker limit.
- Next-time guidance: run focused wrapper integration before the full aggregate gate; `npm run check` currently exposes unrelated baseline lint, TypeScript, and coverage failures.
