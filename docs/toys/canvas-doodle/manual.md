# CANV1 — Canvas Doodle

## What this toy does

Canvas Doodle generates a small layered 2D doodle from a few JSON parameters. It is useful for testing the canvas presenter and experimenting with basic size and color inputs.

This toy does not render arbitrary user-provided shape lists. It generates a fixed doodle pattern from the parameters you provide.

## Input format

Paste a JSON object with any of these optional fields:

- `width`
- `height`
- `background`
- `accent`

```json
{
  "width": 320,
  "height": 180,
  "background": "#f8f6f2",
  "accent": "#1f2937"
}
```

## Defaults

If input is empty or malformed, the toy uses:

```json
{
  "width": 320,
  "height": 180,
  "background": "#f8f6f2",
  "accent": "#1f2937"
}
```

## How the doodle is generated

The toy produces five shapes:

1. a background rectangle covering the full canvas
2. a horizontal color band near the top
3. a left circle
4. a right circle
5. a bottom accent line

The band and circles use generated hue values. The `accent` value controls the bottom line color.

## Output

The toy returns a canvas payload:

```json
{
  "width": 320,
  "height": 180,
  "shapes": []
}
```

The `shapes` array is generated by the toy and rendered by the canvas presenter.

## How to use it

1. Submit empty input to see the default doodle.
2. Change `width` and `height` to resize the canvas.
3. Change `background` to alter the base color.
4. Change `accent` to alter the bottom line.
5. Click **Submit** again.

## Examples

### Dark background

```json
{
  "width": 360,
  "height": 220,
  "background": "#111827",
  "accent": "#f9fafb"
}
```

### Small thumbnail

```json
{
  "width": 180,
  "height": 120,
  "background": "#fef3c7",
  "accent": "#92400e"
}
```

### Wide banner

```json
{
  "width": 640,
  "height": 180,
  "background": "#ecfeff",
  "accent": "#164e63"
}
```

## Tips

- Use standard CSS color strings for `background` and `accent`.
- Keep width and height positive and moderate.
- Use this toy as a simple presenter test rather than a full drawing tool.
- If you want custom shape rendering, this toy would need to be extended to consume a user-provided `shapes` array.

## Troubleshooting

### My custom shapes did not appear

That is expected. The current toy ignores user-provided `shapes` and generates its own fixed composition.

### The colors changed unexpectedly

The band and circles use a generated hue. Only `background` and `accent` are directly controlled by input.

### The doodle looks cramped

Increase `width` and `height`. The generated positions and sizes scale with the canvas dimensions.
